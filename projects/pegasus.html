Pegasus: High-Performance Trading System Implementation

Pegasus is a high-performance limit order book system designed for financial trading applications with extensive multi-threading support. This blog post details the implementation of the system and its various approaches to multi-threading.

Core Features

  • Price-time priority matching algorithm
  • Support for multiple order types (limit, market, IOC)
  • Multi-threading support for processing orders across different symbols
  • Atomic operations for thread safety
  • Optimized for low-latency trading operations

System Architecture

The Pegasus system implements a sophisticated limit order book with an emphasis on performance. The core components include:

  • Order Book: Maintains buy and sell orders sorted by price and time priority
  • Order Manager: Handles the processing of new orders and cancellations
  • Matching Engine: Pairs buy and sell orders according to price-time priority
  • Multi-threading Framework: Enables parallel processing of orders

Multi-threading Implementations

Thread-per-Symbol (OrderBookManager)

This implementation dedicates a separate thread to each trading symbol, allowing for complete parallel processing across different symbols:

  • Each symbol has its own thread and order book instance
  • Orders are routed to their respective symbol threads
  • No contention between different symbols
  • Scales well with increasing number of symbols

Thread Pool Approach

The thread pool implementation provides a more resource-efficient approach:

  • Maintains a fixed pool of worker threads
  • Orders are queued and processed by available threads
  • Better resource utilization compared to thread-per-symbol
  • Configurable thread count based on system capabilities

Basic Multi-threading

A simpler threading model that focuses on direct thread management:

  • Creates threads on-demand for order processing
  • Simpler implementation but less optimal for high-throughput scenarios
  • Useful for systems with limited concurrent symbols

Atomic Operations

To ensure thread safety without excessive locking:

  • Uses atomic operations for critical shared data
  • Reduces lock contention while maintaining data integrity
  • Implemented for order IDs, sequence numbers, and other shared counters

Performance Considerations

The Pegasus system addresses several key performance aspects:

  • Memory Management: Careful allocation and reuse of order objects to minimize garbage collection
  • Lock Contention: Minimized through the use of atomic operations and symbol isolation
  • Cache Locality: Data structures designed to maximize CPU cache efficiency
  • Scalability: Different threading models to adapt to various hardware configurations

ETH/USD Order Book Simulation

The system includes a simulation of an ETH/USD cryptocurrency order book:

  • Realistic price movements and order flow patterns
  • Configurable order rate and market volatility
  • Performance metrics collection for system evaluation
